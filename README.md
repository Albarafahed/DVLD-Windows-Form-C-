# DVLD - Drivers, License, and Vehicle Department System

A professional-grade desktop application designed to streamline the management of driving licenses and vehicle department operations. This project demonstrates advanced software engineering principles including **3-Tier Architecture** and **SOLID** principles using the **C# .NET** ecosystem.

## 🚀 Key Modules & Functionality
- **Driving License Management:** Handling Local & International licenses (Issue, Renew, Replace, Detain).
- **Comprehensive Testing System:** Integrated modules for Vision, Written, and Practical tests.
- **Application Workflow:** Automated tracking of application statuses and types.
- **User & Role Management:** Secure authentication system with detailed permission levels.
- **Advanced People Management:** Centralized person records with image handling and validation.

## 🛠️ Technical Stack
- **Language:** C#
- **Platform:** .NET Framework (WinForms)
- **Database:** SQL Server (Complex Relational Design)
- **Data Access:** ADO.NET (Optimized Queries & Procedures)
- **Version Control:** Git

## 🏛️ Software Architecture
The system is built using a **3-Tier Architecture** to ensure maintainability and scalability:
1.  **Presentation Layer (UI):** Built with WinForms, focusing on user responsiveness and intuitive navigation.
2.  **Business Logic Layer (BLL):** Handles data validation, business rules, and core application logic.
3.  **Data Access Layer (DAL):** Manages all database interactions, ensuring a clean separation from the UI.

## 🔧 Core Principles Applied
- **SOLID Principles:** Applied to ensure clean, reusable, and testable code.
- **Data Integrity:** Implemented robust database constraints and transaction-safe logic.
- **Clean Code:** Focused on naming conventions and organized folder structures for better collaboration.

## 📂 Project Structure
- `/DVLD_Presentation`: The UI forms and resources.
- `/DVLD_Business`: Business logic and entity models.
- `/DVLD_DataAccess`: SQL connectivity and data persistence logic.
- `/Database`: Contains the SQL scripts to rebuild the database.
